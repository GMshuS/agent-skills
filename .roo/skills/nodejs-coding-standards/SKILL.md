# 何时使用
- 当进行Node.js编程时
- 当需要确保代码符合最佳实践时
- 当需要统一团队编码风格时

# 何时不使用
- 当编写非JavaScript代码时
- 当项目已有特定编码规范时

# 核心规范摘要
1. 使用ESLint进行代码检查
2. 遵循模块化设计原则
3. 错误处理使用try/catch或Error-first回调
4. 异步操作优先使用async/await
5. 遵循标准目录结构
