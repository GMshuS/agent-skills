---

# 当使用此技能时

当用户提供每日工作内容记录，需要整理生成周报时使用此技能。适用于：
- 用户输入了一周的工作日志或零散的工作记录
- 需要将工作内容按指定模板格式整理成周报
- 需要区分"本周完成情况"和"下周计划"两部分内容

# 当不使用此技能时

- 用户需要生成其他类型的报告（如月报、日报）
- 用户没有提供具体的工作内容输入
- 用户需要的是实时对话而非报告生成

# 输入要求

用户需要提供：
1. **本周工作内容**：每日完成的具体工作任务
2. **下周工作计划**：计划完成的工作任务列表
3. **时间范围**（可选）：用于生成文件名，如"2024-01-15 至 2024-01-19"，如不提供则自动根据当前日期推算

# 工作流程

## 1. 收集用户输入

向用户询问或等待用户提供：
- 本周已完成的工作内容（按日期或类别组织）
- 下周计划完成的工作任务

## 2. 分类整理工作内容

将用户输入的内容按以下规则整理：

### 本周完成情况分类规则：
- **同类合并**：将同一项目/任务的工作内容合并在一起
- **编号排序**：使用数字编号（1、2、3...）列出具体完成项
- **动词开头**：每条以"完成"、"进行"、"修改"、"配置"等动词开头
- **内容紧凑**：同一任务的多个子项在同一行内，用分号分隔

### 下周工作计划分类规则：
- 保留任务编号（如 ZMCPWORK-58、ZNB-19671 等）
- 简要描述任务内容
- 使用数字编号列出

## 3. 应用周报模板

按照以下格式生成周报（纯文本格式，内容紧凑）：

```
本周工作完成情况

一、[主要项目/类别名称]
1、[任务内容不空行，紧凑格式]；
2、[任务内容不空行]；
3、[任务内容不空行]；

二、[其他工作类别]
1、[任务内容不空行]；
2、[任务内容不空行]；

...

下周工作计划：
1、[任务编号] 任务描述；
2、[任务编号] 任务描述；
...
```

### 纯文本格式规则：
- **文件扩展名**：使用 `.txt` 而非 `.md`
- **任务内容不空行**：每个任务的详细说明不空行，但明细直接要换行
- **任务之间换行**：不同任务类别之间用空行分隔
- **内容紧凑**：去除多余的空白行和 Markdown 格式符号
- **列表格式**：使用"1、"、"2、"等中文编号

## 4. 任务编号完整说明规则

对于包含任务编号（如 ZNB-20899、ZMCPWORK-58 等）的工作项：
- **完整保留中文说明**：任务编号后的中文描述必须完整，不能省略关键信息
- **保持原有格式**：任务编号与中文说明之间用空格分隔
- **示例**：
  - ✅ 正确：`ZNB-20899 鑫智能打新 - 申购预清算后，检查虚拟委托表 simulate_order，部分申购数据的 fund 不为 0`
  - ❌ 错误：`ZNB-20899 申购预清算问题`

## 5. 保存周报文件

### 文件保存规则：
- **保存路径**：`./weekly-report/` 文件夹（相对于当前工作目录）
- **文件命名**：`weekly-report_YYYYMMDD-YYYYMMDD.txt`
  - 格式：`weekly-report_开始日期 - 结束日期.txt`
  - 日期格式：YYYYMMDD（如 20240115）
- **文件扩展名**：使用 `.txt` 纯文本格式
- **文件内容**：完整的周报内容，使用纯文本格式（非 Markdown）

### 文件操作步骤：
1. 检查 `./weekly-report/` 文件夹是否存在，不存在则创建
2. 根据时间范围生成文件名
3. 将周报内容写入文件
4. 向用户确认文件已保存及保存路径

## 6. 输出并确认

- 将生成的周报呈现给用户
- 显示已保存的文件路径
- 询问用户是否需要调整格式或补充内容

# 示例

```
周一：完成了数据 - 融资融券功能迁移
周二：做了数据 - 机构持股功能迁移
周三：在进行数据 - 高管持股功能迁移
周四：配置了 VSCode 的 QT 开发环境
周五：协助测试人员答疑，支持鸿蒙版测试

下周要做：
- 继续协助测试
- 鸿蒙版一期客户端迁移
- Dispatcher 升级 AMQ
- 申购预清算问题 ZNB-20899
- 策略易告警优化 ZNB-21545
```

## 生成的周报（保存到 weekly-report/weekly-report_20260407-20260414.txt）：
```
本周工作完成情况

一、鸿蒙版客户端：行情咨询功能迁移
1、完成"数据 - 融资融券"功能迁移；2、完成"数据 - 机构持股"功能迁移；3、进行"数据 - 高管持股"功能迁移；

二、在 VSCode 中配置 QT 开发与编译环境，方便使用 RooCode、OpenCode 等 AI 编程工具

三、ZMCPWORK-58 开发人员协助测试人员测试：临时答疑、分析测试问题
1、测试与开发答疑；2、鸿蒙版客户端测试支持；

下周工作计划：
1、ZMCPWORK-58 开发人员协助测试人员测试：临时答疑、分析测试问题；
2、ZNB-20899 鑫智能打新 - 申购预清算后，检查虚拟委托表 simulate_order，部分申购数据的 fund 不为 0；
3、ZNB-21545 生产运行 - 策略易中签缴款不足等业务返回错误被定义为 fatal 错误，导致告警，建议优化。
```

# 注意事项

1. **保持格式一致**：使用中文标点符号，编号使用"1、"格式
2. **分号结尾**：每个列表项以分号（；）结尾
3. **任务编号完整**：如 ZNB-20899、ZMCPWORK-58 等编号后的中文说明必须完整保留
4. **语言简洁**：每条工作内容简明扼要，突出完成状态
5. **分类合理**：相关工作内容归入同一类别下
6. **文件保存**：周报自动保存到 ./weekly-report/ 文件夹，使用 .txt 纯文本格式
7. **内容紧凑**：每个任务的内容在同一行内，不换行；任务之间用空行分隔
